# Topic Discovery Framework

## Demand + needs mapping

- Start from keyword demand and user jobs-to-be-done.
- Group opportunities by intent stage (awareness, comparison, decision, retention).
- Separate brand vs non-brand and short-tail vs long-tail.

## Cluster architecture

- Define one pillar topic owner per core business theme.
- Add supporting topics for questions, use cases, comparisons, and objections.
- Avoid overlap where multiple pages target the same core intent.

## Prioritization score

Evaluate each topic by:

- business relevance
- traffic potential
- ranking feasibility
- conversion proximity
- production effort
