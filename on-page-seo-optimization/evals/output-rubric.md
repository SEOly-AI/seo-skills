# On-Page Output Rubric

Use this rubric before final delivery. Mark each as **Pass / Partial / Fail**.

## A. Input and Context Quality (20%)

- [ ] Context files were checked and availability reported.
- [ ] Page type, optimization baseline, competitors, and conversion goal were confirmed.
- [ ] Questions focused only on missing data.

## B. Metadata and Structure (25%)

- [ ] Title/meta are intent-aligned and CTR-oriented.
- [ ] Heading hierarchy is clear and logically structured.
- [ ] Canonical/indexability considerations are included.

## C. Content and E-E-A-T (25%)

- [ ] Recommendations increase depth, clarity, and relevance.
- [ ] E-E-A-T reinforcement is explicit and practical.
- [ ] Media optimization actions are specific.

## D. Linking and UX (20%)

- [ ] Internal linking actions support intent and authority flow.
- [ ] External references improve credibility where appropriate.
- [ ] UX/CTA improvements align with query stage and conversion goals.

## E. Actionability (10%)

- [ ] Output includes prioritized, page-level actions (P0/P1/P2).
- [ ] Deliverables are executable without ambiguity.

## Hard Fail Conditions

- Generic recommendations with no page-level specificity.
- Missing conversion alignment.
- No distinction between primary and supporting intent elements.
