# Technical SEO Output Rubric

Use this rubric before final delivery. Mark each as **Pass / Partial / Fail**.

## A. Input and Scope Quality (20%)

- [ ] Site type, scale, architecture, and access constraints are clear.
- [ ] Recent changes/migrations are documented.
- [ ] Priority modules are justified.

## B. Crawl/Index Diagnosis (25%)

- [ ] Crawl waste and index bloat are quantified.
- [ ] Signal conflicts (robots/meta/canonical/sitemap) are identified.
- [ ] Must-index vs non-index governance is actionable.

## C. Architecture/URL/Redirect Quality (20%)

- [ ] Internal structure supports strategic pages.
- [ ] URL normalization rules are clear.
- [ ] Redirect strategy avoids chains/loops and preserves intent.

## D. Performance and Mobile (20%)

- [ ] CWV issues are mapped to technical causes.
- [ ] Mobile parity and usability issues are addressed.
- [ ] Fix plan includes validation checkpoints.

## E. Execution and Measurement (15%)

- [ ] Prioritized implementation backlog (P0/P1/P2) is included.
- [ ] Ownership and timelines are explicit.
- [ ] KPI monitoring framework is measurable.

## Hard Fail Conditions

- No crawl/index signal reconciliation.
- Recommendations without implementation details.
- No rollback safeguards for migration-sensitive changes.
