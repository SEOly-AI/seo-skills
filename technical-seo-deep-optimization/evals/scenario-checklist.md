# Scenario Checklist

## S1 - Large Site Crawl Budget Constraints

- [ ] Crawl budget allocation model is defined.
- [ ] Low-value crawl paths are controlled.
- [ ] High-value page discovery/indexing is improved.

## S2 - Site Migration or Replatforming

- [ ] Redirect map and canonical strategy are validated.
- [ ] Pre/post migration QA checklist is complete.
- [ ] Rollback and monitoring plan is documented.

## S3 - Ecommerce Technical SEO

- [ ] Dynamic URL and faceted navigation controls are implemented.
- [ ] Variant duplication and out-of-stock policy are handled.
- [ ] Product/category crawl priorities are clear.

## S4 - SaaS Technical SEO

- [ ] Login wall impact on crawl/index is addressed.
- [ ] SPA rendering/indexability risks are mitigated.
- [ ] SEO pages and app routes are technically separated.

## S5 - Ranking Drop from Technical Regressions

- [ ] Root cause category is identified with evidence.
- [ ] Immediate containment actions are proposed.
- [ ] Recovery milestones and KPIs are defined.
