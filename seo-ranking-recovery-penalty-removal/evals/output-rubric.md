# Ranking Recovery Output Rubric

Use this rubric before final delivery. Mark each as **Pass / Partial / Fail**.

## A. Incident Framing Quality (20%)

- [ ] Decline scope and timeline are clearly defined.
- [ ] Impact segmentation is complete.
- [ ] Inputs and data limits are transparent.

## B. Diagnosis Quality (25%)

- [ ] Algorithmic/self-inflicted/external causes are separated.
- [ ] Evidence supports each hypothesis.
- [ ] Confidence levels are assigned appropriately.

## C. Root Cause and Remediation (25%)

- [ ] Root causes are specific and actionable.
- [ ] Penalty removal plan addresses link/content/technical risks.
- [ ] Remediation sequence is prioritized and realistic.

## D. Recovery Plan Quality (20%)

- [ ] Recovery actions are owner-assigned and time-bounded.
- [ ] Monitoring checkpoints are defined.
- [ ] KPI recovery targets are measurable.

## E. Prevention Controls (10%)

- [ ] Anti-regression controls are documented.
- [ ] Ongoing audit and alert system is specified.

## Hard Fail Conditions

- No evidence-backed diagnosis.
- Generic recommendations without incident-level specificity.
- No prevention controls for recurrence risk.
