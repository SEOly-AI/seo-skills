# GEO/AI SEO Output Rubric

Use this rubric before final delivery. Mark each as **Pass / Partial / Fail**.

## A. Context Quality (20%)

- [ ] Read product marketing context files and reported availability.
- [ ] Confirmed target AI platforms, content status, schema status, and tracking access.
- [ ] Asked only gap-focused questions.

## B. Intent Alignment (20%)

- [ ] Intent clusters are explicit and mapped to page ownership.
- [ ] Intent-to-page conflicts are identified and resolved.
- [ ] AI retrieval patterns are reflected in content plan.

## C. GEO Content Plan (25%)

- [ ] Provides concrete content changes (structure, summary blocks, evidence, entity clarity).
- [ ] Avoids generic AI-writing patterns and low-information prose.
- [ ] Keeps traditional SEO compatibility.

## D. Schema Plan (20%)

- [ ] Schema recommendations align with page intent/type.
- [ ] Required properties and validation approach are included.
- [ ] No over-markup or irrelevant schema stacking.

## E. Measurement Plan (15%)

- [ ] Includes AI visibility KPIs and classic SEO KPIs.
- [ ] Defines iteration loop and reporting cadence.
- [ ] Connects performance metrics to business outcomes.

## Hard Fail Conditions

- Invented platform/data access capabilities.
- No clear mapping between intent clusters and page owners.
- Non-actionable recommendations without prioritization.
