# Deliverable Quality Rubric (Self-check)

Use this before final delivery. Mark each item as **Pass / Partial / Fail**.

## A. Context and Scope (20%)

- [ ] Attempted to read `.agents/product-marketing-context.md` and `.claude/product-marketing-context.md` (and noted availability).
- [ ] Site type, scope, competitors, and tool permissions are confirmed or clearly marked as unknown/TBD.
- [ ] No repeated questions for information already present in context.

## B. Keyword Discovery (25%)

- [ ] Output is organized as clusters, not a flat keyword dump.
- [ ] No fabricated exact search volumes when tools are unavailable.
- [ ] Seed sources are traceable (product/category/user tasks/competitors/GSC).

## C. Dimension Analysis (20%)

- [ ] Intent labels are consistent with `references/intent-and-dimensions.md`.
- [ ] Competitor layout analysis is path/type-level, not vague commentary.

## D. On-site Mapping (20%)

- [ ] Each cluster/core term has a clear primary URL owner (or planned URL with note).
- [ ] Mapping columns align with `references/mapping-templates.md` or pass script validation.
- [ ] Supporting page roles are clearly defined.

## E. Cannibalization Audit (15%)

- [ ] Includes conflict table or explicit no-conflict conclusion with evidence.
- [ ] Recommendations are executable (merge, 301, title/H1 changes, internal links), not generic advice.

## Hard Fail Conditions

- Inventing business facts not provided by the user.
- Inventing exact search metrics without data access.
- Assigning multiple equal primary URLs to one core keyword without clear ownership.
