# Scenario Coverage Checklist

Use this list to verify scenario completeness before delivery.

## S1 — New Site (0 -> 1)

- [ ] Keyword clusters align with proposed IA/site sections.
- [ ] Priority model (P0-P3) and resource assumptions are explicit.
- [ ] Preventive cannibalization guidance is included (e.g., blog vs product ownership).

## S2 — Existing Site Optimization

- [ ] Current ranking URLs are mapped to target keywords.
- [ ] Cannibalization list includes URL-level conflicts and priority.
- [ ] Roadmap is split into immediate / short-term / long-term actions.

## S3 — New Product / New Business Line

- [ ] Incremental keywords are separated from existing core terms.
- [ ] Recommended new page types are defined (product/category/comparison/support/etc.).
- [ ] Internal linking strategy to transfer authority from existing pages is included.

## S4 — Traffic Plateau Breakthrough

- [ ] Clearly identifies topics outside the current keyword system.
- [ ] Each opportunity cluster is mapped to a page type and coverage gap reason.
- [ ] Includes measurable next steps (monitoring groups, post-launch checks).
